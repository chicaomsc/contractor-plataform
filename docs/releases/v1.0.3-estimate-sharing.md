# Release v1.0.3 — Estimate Sharing

**Data:** 2026-07-18
**Sprint:** 10D — Estimate Sharing
**Escopo:** link público, revogável e com expiração para visualizar/baixar o PDF de um orçamento sem autenticação, com botões de partilha (copiar link, WhatsApp) no dashboard e auditoria mínima de acesso.

---

## 1. Plano Executado

Contratos reais confirmados antes de codar: `SecurityConfig` já cobre `/public/**` (nenhuma alteração necessária), `/uploads/**` já é público (logo funciona sem mudanças), `EstimatePdfDocumentFactory`/`EstimatePdfService` já produzem um modelo 100% formatado reaproveitável tanto para o PDF quanto para a view JSON pública, e o padrão de controlador público (`PublicSiteController`/`PublicServiceController`) foi replicado tal-qual. Implementação seguiu: modelagem → migration → geração/hash de token → serviço → controladores privado/público → testes → frontend privado → página pública → E2E → documentação.

---

## 2. Modelagem

Nova entidade `EstimateShare` (`estimate.domain`), independente do agregado `Estimate` — referenciado por `estimateId`, nunca modifica `Estimate`:

```
EstimateShare
├── id: UUID
├── companyId: UUID          -- denormalizado, para isolamento direto sem join
├── estimateId: UUID
├── tokenHash: String(64)    -- SHA-256 hex do token bruto; único
├── expiresAt: Instant
├── revokedAt: Instant?      -- null enquanto ativo
├── lastAccessAt: Instant?
├── accessCount: long
├── createdByUserId: UUID
├── createdAt / updatedAt    -- herdados de BaseEntity
```

Migration **V10** (`create_estimate_shares_table.sql`): FK `estimate_id → estimates(id) ON DELETE CASCADE` — um `Estimate` só pode ser deletado fisicamente em `DRAFT` (regra já existente da Sprint 10A); o cascade garante que apagar o orçamento invalida automaticamente qualquer link, sem código de aplicação adicional. `UNIQUE (token_hash)`. Índices por `estimate_id`/`company_id`.

**Modelo:** no máximo um link ativo por orçamento. Criar um novo (`POST /estimates/{id}/share`) revoga automaticamente o anterior — mesmo comportamento de "gerar novo link" de produtos como Google Docs.

## 3. Estratégia do Token

`SecureTokenGenerator.generate()` (`common/security`): 20 bytes de `SecureRandom`, Base64 URL-safe sem padding → 160 bits de entropia (acima do mínimo de 128 exigido), string opaca de 27 caracteres, sem relação com nenhum ID interno. Nunca sequencial, nunca UUID.

Detalhes completos e alternativas rejeitadas em **[ADR-009](../adr/ADR-009-estimate-share-token-strategy.md)**.

---

## 4. Estratégia de Segurança

- **Hash, não texto plano:** `TokenHasher.sha256Hex` — só o hash é persistido (`tokenHash`). Desvio deliberado do precedente `RefreshToken` (texto plano), justificado em ADR-009 pela exposição muito maior de um link público partilhado.
- **Token mostrado uma única vez:** `POST /estimates/{id}/share` é o único endpoint que devolve o token bruto. `GET /estimates/{id}/share` devolve apenas estado (status/datas/contagem), nunca o token — o backend simplesmente não tem como recuperá-lo.
- **Resposta uniforme para token inexistente/expirado/revogado:** sempre 404, nunca uma variação que confirme "quase acertou" a um atacante enumerando tokens.
- **Isolamento multi-tenant:** toda operação privada valida `(estimateId, companyId)` via `EstimateRepository.findByIdAndCompanyId` antes de tocar em qualquer `EstimateShare`. A rota pública nunca recebe nem aceita `companyId`/`estimateId` do cliente — resolve ambos exclusivamente a partir do `EstimateShare` já validado pelo token.
- **Nenhum vazamento de dado interno:** `PublicEstimateShareResponse` é um DTO construído campo a campo a partir do `EstimatePdfDocument` já sanitizado — nunca serializa a entidade `Estimate`, `Company` ou `EstimateShare` diretamente.

## 5. API Privada (autenticada, apenas OWNER)

```
POST   /estimates/{id}/share   → cria (ou regenera, revogando o anterior) o link; único momento em que `token` vem preenchido
GET    /estimates/{id}/share   → status do link atual (ACTIVE | EXPIRED | REVOKED), sem token
DELETE /estimates/{id}/share   → revoga o link ativo
```

Expiração: `expiresInDays` opcional no corpo do `POST` (`1`–`365`), default **30 dias** — nunca infinito por padrão, mesmo quando o campo não é enviado.

---

## 6. API Pública (sem autenticação)

```
GET /public/share/{token}        → PublicEstimateShareResponse (JSON)
GET /public/share/{token}/pdf    → mesmo pipeline de PDF da rota autenticada
```

`PublicEstimateShareResponse` expõe exclusivamente: empresa (nome, logo, contactos, morada — os mesmos já considerados públicos no PDF desde a Sprint 10C), cliente (apenas nome), itens, materiais, resumo financeiro (já formatado), status/rascunho/cancelado, observações, condições. **Nada administrativo, nenhum `companyId`, nenhum `estimateId`, nenhum ID interno** — verificado por teste de integração que faz asserção negativa sobre o corpo da resposta.

---

## 7. Frontend

`ShareEstimatePanel.tsx`, integrado em `/dashboard/estimates/[id]` logo abaixo do resumo financeiro:

- **Sem link ativo:** botão "Compartilhar".
- **Link recém-criado (token em memória):** campo somente-leitura com o link completo, botão "Copiar link" (com feedback "Copiado" por 2s), botão "WhatsApp".
- **Link ativo mas token não disponível** (ex.: após recarregar a página): estado "Link ativo · expira em X · acedido N vezes", com "Gerar novo link" e "Revogar" — nunca tenta reexibir um token que o backend não devolve.
- **Revogar:** invalida o link imediatamente, refletido também na rota pública.

---

## 8. Página Pública

Rota `/share/[token]` (`frontend/src/app/share/[token]/page.tsx`), fora de `/dashboard` — **não** passa pelo `middleware.ts` (matcher é `/dashboard/:path*`), logo é acessível sem sessão. `robots: { index: false, follow: false }` — não indexável. Layout limpo: cabeçalho com logo/nome da empresa, número/título/status do orçamento, dados do cliente, tabelas de serviços/materiais, resumo financeiro, observações/condições, rodapé com contactos da empresa, botão "Baixar PDF". Token inexistente/expirado/revogado mostra uma mensagem de indisponibilidade, nunca um erro genérico ou stack trace.

---

## 9. Compartilhamento WhatsApp

Nenhuma integração com a API do WhatsApp Business. `buildEstimateShareWhatsAppHref` (`dashboard/utils/estimate-share-links.ts`) apenas constrói `https://wa.me/?text=...` com o link público codificado — abre o WhatsApp (app ou web) com a mensagem pré-preenchida, deixando o utilizador escolher o destinatário.

---

## 10. Download PDF

**Reaproveita integralmente o pipeline da Sprint 10C** — nenhum renderer novo. `PublicEstimateShareController.downloadSharedPdf` resolve `(companyId, estimateId)` a partir do `EstimateShare` validado e chama `EstimatePdfService.generatePdf(companyId, estimateId)`, o mesmo método já usado por `GET /estimates/{id}/pdf`. Mesmos headers (`Content-Type: application/pdf`, `Content-Disposition` com filename sanitizado).

---

## 11. Auditoria

Apenas `createdAt` (herdado), `lastAccessAt` e `accessCount`, incrementados a cada acesso público bem-sucedido (view **e** download contam como acesso). **Nenhum sistema de analytics** — sem IP, user-agent, geolocalização ou qualquer telemetria além dos três campos pedidos.

## 16. Riscos Adiados

- **Token não recuperável após a criação** — decisão consciente (ADR-009); recuperação é sempre via regeneração, o que revoga o link anterior.
- **Sem cache do PDF/JSON público** — cada acesso reprocessa a partir dos dados persistidos; aceitável para o volume do MVP, mesmo padrão já aceite na Sprint 10C.
- **Sem analytics de acesso** — apenas os três campos de auditoria mínima pedidos; um painel de "quem acedeu quando" fica fora do escopo.
- **Sem rate limiting dedicado nas rotas públicas de partilha** — mesma postura do restante da API pública (`/public/sites/**`) nesta fase do MVP; reavaliar junto com o rate limiting geral já listado no roadmap (Sprint 12).
- **Um único link ativo por orçamento** — suficiente para o caso de uso atual (reenviar ao mesmo cliente); múltiplos links simultâneos por orçamento fica fora do escopo.
