# v0.9.3 — MVP Hardening

**Data:** 2026-07-16  
**Sprint:** 9 — MVP Hardening

## Plano Executado

- E2E Playwright adicionado com fluxo principal e smokes críticos.
- Revisão de sessão/autenticação documentada.
- Testes backend cross-tenant adicionados.
- Política explícita de upload implementada.
- CORS revisado sem wildcard com credenciais.
- Security headers adicionados no Next.js e Spring Security.
- Build removido de dependência obrigatória de Google Fonts.
- Dashboard mobile corrigido para expor logout no drawer.
- Rota experimental `_lab/before-after` removida do bundle.
- CI atualizado com smoke E2E no frontend e package build no backend.

## Correções

- `ImageUploadPolicy` bloqueia SVG, MIME inválido, extensão incompatível, arquivo vazio, tamanho acima de 5 MB e conteúdo disfarçado.
- `LocalStorageService` rejeita path traversal em escrita e exclusão.
- `/uploads/**` recebe cache público de 30 dias.
- `UploadArea` aceita apenas PNG, JPEG e WebP.
- `DashboardShell` mostra logout no sidebar/drawer, corrigindo acesso mobile.
- `next/font/google` removido; tokens usam fontes de sistema.
- Vitest exclui `e2e/**`, separando unit tests de Playwright.

## E2E

- `e2e/smoke.spec.ts`: landing, login inválido, proteção de dashboard, tenant inexistente, logout e bloqueio após logout.
