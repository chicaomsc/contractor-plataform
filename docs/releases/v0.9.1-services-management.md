# Release v0.9.1 — Services Management

**Data:** 2026-07-16  
**Sprint:** 8B — Services Management  
**Escopo:** gestão administrativa de serviços

---

## Objetivo

Implementar a área administrativa completa para gerenciamento dos serviços oferecidos pela empresa, consumindo exclusivamente a API existente do Spring Boot e mantendo a landing pública como reflexo dos dados persistidos.

Esta release não altera backend, endpoints, autenticação, ViewModels públicos, arquitetura ou regras de negócio.

## Funcionalidades Administrativas

- Listagem de serviços em `/dashboard/services`
- Criação de serviço
- Edição de serviço
- Exclusão com confirmação explícita
- Ativação e desativação de serviço
- Ordenação por mover para cima e mover para baixo
- Estado vazio para empresas sem serviços cadastrados
- Feedback de loading, saving, success, error e retry

Drag and drop não foi implementado porque não havia infraestrutura existente no projeto. A ordenação usa controles explícitos de mover para cima e mover para baixo.

---

## Formulário

Campos implementados com React Hook Form e Zod:

- Nome
- Descrição curta
- Descrição
- Ícone
- Ordem
- Status

Validações:

- Nome obrigatório
- Limites de tamanho compatíveis com o contrato administrativo
- Ordem numérica opcional, sem valores negativos
- Status booleano

---

## Integração com Backend

Endpoints existentes consumidos:

- `GET /services`
- `POST /services`
- `PUT /services/{id}`
- `DELETE /services/{id}`
- `PATCH /services/{id}/reorder`

Não foram criados endpoints, migrations, mocks permanentes, Route Handlers ou Server Actions.

---

## Reflexo na Landing Pública

A landing pública continua consumindo os endpoints públicos já existentes. Alterações feitas no dashboard afetam a landing indiretamente pelo backend, sem duplicar regra de negócio no frontend.

Serviços desativados deixam de aparecer na landing conforme o comportamento existente da API pública.
