# Release v0.9.2 — Gallery Management

**Data:** 2026-07-16  
**Sprint:** 8C — Gallery Management  
**Escopo:** gerenciamento administrativo da galeria

---

## Objetivo

Implementar um Media Manager simples, consistente e reutilizável para administrar as imagens utilizadas na landing pública, consumindo exclusivamente a API existente do Spring Boot.

Esta release não altera backend, endpoints, autenticação, arquitetura, ViewModels públicos ou regras de negócio.

## Funcionalidades Administrativas

- Listagem de itens em `/dashboard/gallery`
- Criação de item de galeria
- Edição de metadados
- Exclusão com confirmação
- Upload de imagem before
- Upload de imagem after
- Remoção de imagem before/after
- Preview de pares before/after
- Ativação e desativação
- Definição e remoção de destaque
- Ordenação por mover para cima e mover para baixo
- Estado vazio para empresas sem imagens cadastradas
- Estados de loading, saving, uploading, success, error e retry

O modelo de pares before/after usa exclusivamente o contrato existente do backend: um item de galeria possui `beforeImageUrl` e `afterImageUrl`. Nenhuma lógica nova de associação foi criada.

---

## Upload

O upload usa `multipart/form-data` nos endpoints existentes.

Validações no frontend:

- arquivo deve ser imagem (`image/*`);
- tamanho máximo de 5 MB;
- arquivo inválido exibe erro local antes de chamar a API.

O frontend mostra progresso do fluxo de envio e feedback de sucesso/erro. Não há crop, editor, compressão, filtros ou processamento de imagem.

---

## Integração com Backend

Endpoints existentes consumidos:

- `GET /gallery`
- `POST /gallery`
- `PUT /gallery/{id}`
- `DELETE /gallery/{id}`
- `PATCH /gallery/{id}/feature`
- `PATCH /gallery/{id}/reorder`
- `POST /gallery/{id}/before-image`
- `POST /gallery/{id}/after-image`
- `DELETE /gallery/{id}/before-image`
- `DELETE /gallery/{id}/after-image`

Não foram criados endpoints, migrations, mocks permanentes, Route Handlers ou Server Actions.

---

## Reflexo na Landing Pública

A landing pública continua consumindo os endpoints públicos já existentes. Alterações feitas no dashboard afetam a landing indiretamente pelo backend, sem duplicar regra de negócio no frontend.

Itens desativados deixam de aparecer na landing conforme o comportamento existente da API pública. Itens destacados continuam ordenados pelo backend público conforme o contrato existente.
