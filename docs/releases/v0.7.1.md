# Sprint 7B.5 — Public Site Integration

## Objetivo

Validar integralmente a integração entre o frontend Next.js e as APIs públicas do Spring Boot antes da implementação visual da landing page.

## Integração

A rota `/` resolve o tenant através de `NEXT_PUBLIC_COMPANY_SLUG` e carrega dados públicos por TanStack Query num boundary client pequeno. O layout raiz, metadata, estados globais e estrutura base permanecem no App Router.

## Contratos públicos

Endpoints consumidos exclusivamente:

- `GET /public/sites/{companySlug}`
- `GET /public/sites/{companySlug}/services`
- `GET /public/sites/{companySlug}/gallery`

## Hooks

Criados:

- `usePublicSite`
- `usePublicServices`
- `usePublicGallery`

As query keys ficam centralizadas em `features/public-site/api/query-keys.ts`.

## Mappers

Separação implementada:

API DTO → Mapper → ViewModel → Component

Mappers:

- `mapPublicSiteDto`
- `mapPublicServiceDto`
- `mapPublicGalleryItemDto`

## Branding

Fluxo:

API branding → validação HEX `#RRGGBB` → tokens permitidos → CSS custom properties

Cores inválidas são descartadas e caem nos tokens de fallback.

## Loading

Preview técnico possui skeletons discretos para site, serviços e galeria. Skeletons são marcados como conteúdo não real e os containers usam `aria-busy`.

## Error states

Coberto:

- slug ausente;
- API indisponível;
- site inexistente;
- resposta inválida;
- falha parcial de serviços;
- falha parcial de galeria.

Erro de site é bloqueante. Erros de serviços ou galeria preservam header/footer e o restante dos dados carregados.

## Empty states

Coberto:

- empresa sem serviços ativos;
- empresa sem itens de galeria;
- empresa sem logo;
- empresa sem footer text;
- galeria sem pares before/after completos.

## Testes

Frontend cobre hooks, mappers, API client, preview técnico, estados de erro, empty states e branding seguro.

Backend cobre contrato público por slug, slug inexistente, isolamento por empresa, serviços/galeria ativos, ordenação, ausência de dados sensíveis, before/after opcionais e slug com acentos.

## Riscos

- O build com `next/font` precisa de acesso de rede para baixar fontes no ambiente de CI.
- `npm audit` ainda reporta vulnerabilidade transitiva em `next/postcss`; a correção automática sugerida pelo npm implica downgrade quebrado e não foi aplicada.

## Pendências

- Resolução de tenant por domínio.
- Conteúdo real e composição visual da landing.
- Validação final de logo, fotografias e paleta do primeiro tenant.

## Critérios para Sprint 7C

- Manter DTOs fora dos componentes.
- Reutilizar hooks e ViewModels existentes.
- Implementar hero, serviços, galeria e demais secções apenas na Sprint 7C.
- Não introduzir backend-sombra no Next.js.
